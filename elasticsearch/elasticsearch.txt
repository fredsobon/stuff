=== notes elasticsearch : ===

- status du cluster :
http://server:9200/_cluster/health?pretty=true

- list des index : 
http://server:9200/_cat/indices?v

- repartition index / sharding :
http://server:9200/_plugin/head/




- Arret provisoire de la replication de sharding : 
on peut le faire directement depuis l'interface principale de kibana

> section dev tools : 

saisir la requette suivante : 

PUT /_cluster/settings
{
    "transient" : {
        "cluster.routing.allocation.enable" : "none"
    }
}

puis cliquer sur la petite flèche verte pour jouer la requette 
-> qui passe dans la fenètre de droite..




Une fois l'intervention faite sur notre / nos serveur(s) , on va pouvoir relancer le redispatch de shard :

avec la même procedure ..mais en changeant de requette : 
PUT /_cluster/settings
{
    "transient" : {
        "cluster.routing.allocation.enable" : "all"
    }
}
