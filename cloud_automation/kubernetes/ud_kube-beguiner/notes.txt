= pres docker : ==

les containers docker partagent  le kernel  mais sont tous des process cloisonnés : isolated containers.
le but est de containeriser les applications.
une image est un template
un container est l'execution d'une image

= container orchestration presentation =
on va utiliser une plaateforme qui va se scaler en fonction de la charge.
kubernetes : est le standart et est supporter par les providers cloud.


= concepts =


node -> worker machine ( appellé avant minion )
cluster ->: set of nodes qui marchent ensemble et qui peut ainsi permettre la ha
master -> node configurer en master et est responsable de l'orchestration des containers sur les nodes 

api-server
kubelet -services : agent tounrant sur les nodes et responsables du bon fonctionnement des containers
controller : cerveau / prend la decision de poper des containers 
sheduler : destribue le travail sur les nodes 
etcd : store clé / valeurs  /stocke le conf 
container runtime : docker par exemple tourne sur les nodes pour executer les containers.
master vs  nodes


kubectl : outil permettant de piloter en ligne de commande.
